# Screenshot Worklog — Product Spec

## Goal

Manage screenshots used for working-time accounting. Recognize the time area in screenshots, save extracted time to JSON or database, and generate statistics/analysis.

## Core flow

1. User uploads or imports screenshots.
2. System stores original screenshot.
3. Config determines where the time block is located.
4. OCR or AI agent extracts time.
5. Result is stored in JSON initially, DB later.
6. Statistics are generated by day/week/month/project.

## Storage

```text
data/
  screenshots/original/
  screenshots/cropped/
  records/worklog.jsonl
  reports/
```

## Config example

```yaml
recognition:
  mode: configurable_region
  region:
    x: 120
    y: 80
    width: 400
    height: 120
  fallback_agent: true
storage:
  initial: jsonl
  future: sqlite|postgres
```
